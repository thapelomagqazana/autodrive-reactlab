# Code of Conduct

## Our Pledge

AutoDrive Lab is a learning, engineering, and portfolio project.

We are committed to making participation in this project a respectful, welcoming, and constructive experience for everyone, regardless of background, experience level, identity, or location.

Everyone who participates in this project is expected to act with professionalism, kindness, and respect.

---

## Our Standards

Examples of behaviour that contributes to a positive project environment include:

- Using welcoming and inclusive language
- Giving constructive feedback
- Respecting different skill levels
- Being patient with beginners
- Asking questions in good faith
- Accepting correction gracefully
- Focusing criticism on code, ideas, and design decisions rather than people
- Helping others learn and improve
- Assuming good intent where reasonable

Examples of unacceptable behaviour include:

- Harassment, intimidation, or threats
- Insults, personal attacks, or mocking contributors
- Discriminatory language or conduct
- Unwanted sexual attention or comments
- Publishing someone’s private information without permission
- Deliberate disruption of discussions, issues, or pull requests
- Trolling or inflammatory comments
- Repeatedly ignoring maintainer guidance
- Any conduct that would reasonably be considered inappropriate in a professional setting

---

## Project Communication

Project discussions should stay focused on:

- Code quality
- Architecture
- Testing
- Documentation
- Security
- Accessibility
- Performance
- Learning and improvement

Disagreement is allowed.

Disrespect is not.

When reviewing code, prefer language such as:

```text
Could we simplify this function?
This may be easier to test if we extract a helper.
I think this belongs in the simulation engine rather than the component.
Can we add a test for this edge case?
```

Avoid language such as:

```text
This is stupid.
You clearly do not understand.
Why would anyone write this?
```

---

## Scope

This Code of Conduct applies to all project spaces, including:

- GitHub issues
- Pull requests
- Discussions
- Code reviews
- Commit comments
- Project documentation
- Any public or private communication related to AutoDrive ReactLab

It also applies when someone is representing the project in public spaces.

## Enforcement

Maintainers may take appropriate action in response to behaviour that violates this Code of Conduct.

Possible actions include:

- Private warning
- Public clarification
- Comment removal
- Temporary participation restriction
- Permanent ban from project spaces
- Reporting serious abuse to the relevant platform

The goal of enforcement is to protect the project community and maintain a professional learning environment.

---

## Maintainer Responsibilities

Maintainers are responsible for:

- Clarifying acceptable behaviour
- Applying this Code of Conduct fairly
- Removing harmful or inappropriate content
- Protecting contributors from harassment
- Encouraging constructive technical discussion

Maintainers should not use their position to insult, intimidate, or silence contributors acting in good faith.

---

## Good Faith Participation

This project welcomes learners.

Mistakes are expected.

Questions are welcome.

Contributors are encouraged to:

- Ask before assuming
- Explain before criticizing
- Teach where possible
- Learn where necessary
- Keep the project focused and respectful
